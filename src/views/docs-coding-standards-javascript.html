JavaScript guidelines

Overview

'use strict'; must be at every page top
'use strict'; //..... your code
Use === and !== over == and !=
//Bad if(a==b && a!=n){ //... } //Good if(a===b && a !==b){ //... }
Use === and !== over == and !=
//Bad if(a==b && a!= n){ //... } //Good if(a===b && a!==b){ //... }
Use readable code
//Bad if (variable !== '') { //.... } //Good if (variable) { //... } //Bad if (name.length > 0) { //.... } //Good if (name.length) { //... }
Use braces { .. }
//Bad if (eq) return false; //Good if (eq) { return false; }

Objects

Use the literal syntax for object creation.
// bad var item = new Object(); // good var item = {};
Multiline object initializers indented 2 spaces
var names = { height: 100, top: 50, width: 200, left: 10 };

Arrays

Use the literal syntax for array creation.
// bad var items = new Array(); // good var items = [];
Single-line array initializers are allowed when they fit on a line.
var arr = [1, 2, 3]; // No space after [ or before ].
Multiline array initializers indented 2 spaces.
// Array initializer. var names = [ 'Vincent', 'Jasper', 'Tom' ];

Strings

Use single quotes ' ' for strings.
//Bad var name = "Jasper"; //Good var name = 'Vincent';

&& and ||

When using if for variable assignment use default opperator ||.
// bad function donation(money){ var donated; if(money){ donated = money; }else{ donated = 'no money'; } } // good function donation(money){ var donated = money || 'no money'; }
Use && to shortening your code.
if(vincent === 'cool'){ if(jasper === 'cooler'){ //Do someting... } } // good if(vincent === 'cool' && jasper === 'cooler'){ //Do someting }
Do not use || and && for function calls.
//Bad scope.$$phase || (scope.$root && scope.$root.$$phase) || scope.$digest(); //Bad attr.bsShow && doSometing();

Operator ?:

Use return and ?: like this.
// bad if (val) { return foo(); } else { return bar(); } // good return val ? foo() : bar();

Properties

Use dot notation when accessing properties without variable.
var vincent = { age: 22, profession: 'developer' }; // bad var profession = vincent['profession']; // good var profession = vincent.profession;

Variables

Always use var to declare variables otherwhise you will create global variables !
//bad coolPerson = 'Jasper'; //good var coolPerson = 'Vincent';
Use one var declaration per variable.
//bad var lola = 18, isBlack = true, bar = 'box'; var lola = 18; var isBlack = true; var bar = 'box';
Each variable has its own line and comment. They should be listed in alphabetical order if possible.
var bar = 'box'; // The name of the bar var isBlack = true; // Is the bar black var lola = 18; // the age of lola

Semicolon

Do not use a semicolon at the end of a if statement.
//Bad if(1===1){ //your code return ''; }; //Good if(1===1){ //your code return ''; }

Comments

Use /** ... */ for multiline comments.
/** * funcName() returns a new element * * @param {paramType} paramName * @return {Type} element */ function funcName(paramName) { // ...stuff... return elem; }
Use // for single line comments. Place single line comments on a newline above the subject of the comment. Put an empty line before the comment.
//Good function getName() { console.log('getting name'); // set the default name to 'john' var name = this.name || 'john'; return name; }
