The visibility classes per breakpoint each come in 3 variations, dependent on the css display property.

Group of classes CSS display
.visible-*-block display: block;
.visible-*-inline display: inline;
.visible-*-inline-block display: inline-block;

For example, for the smallest breakpoint, these will be .visible-xs-inline, .visible-xs-block and .visible-xs-inline-block

Screen reader and keyboard navigation

Developers sometimes use display: none to hide content, but that can be problematic for screenreaders. There is a possibility to use sr-only to hide an element, except for screenreaders, for which they might be vital information. Combine with sr-only-focusable to show the element when it gets focus.

For good measure, you should add the ARIA attribute aria-hidden="true"
